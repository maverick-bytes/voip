# VoIP on UniFiOS

These scripts allow you to configure VoIP service for customers who have replaced their ISP router with a Unifi gateway based on UnifiOS (Cloud Gateways (UCG-XX), Gateways (UXG-XX), Dream routers (UDM-XX), etc...).

Since it is currently impossible to configure via the Unifi GUI a WAN interface with multiple VLANs, as someone would expect from such prosumer products to have, in contrast to 10$ devices and default isp ONTs.. this script creates a new interface and handles VoIP traffic routing to your local network.

Tested on **Unifi Cloud Gateway Fiber (UCG-Fiber)** with UnifiOS.

## Prerequisites

-  A UniFi gateway running UniFiOS (UCG-Fiber, UCG-Max, UDM-Pro, UXG-Pro, etc.)
-  SSH access to the gateway
-  Your ISP's VoIP delivered on a VLAN tag on the same WAN port as internet (common with FTTH IMS/SIP services)
-  A SIP softphone or ATA on your LAN (e.g. MicroSIP, Grandstream, Cisco SPA)

## Installation

As previously mentionned SSH access to the gateway is required.

On the gateway, download this repository into the `/data/voip` directory (the `/data` content is preserved during reboots and firmware upgrades):
```bash
cd /data
curl -sL https://github.com/maverick-bytes/voip/archive/refs/heads/main.tar.gz | tar -xvz
mv voip-main voip
cd voip
```

Make the scripts executable:
```bash
chmod +x voip voipd
```

Install by running :
```bash
cd /data/voip
./voip install
./voip install-ui
```

You can then either open **`<gateway-ip>/voip`** on your browser to configure the VoIP service using the Web UI (simplest and user-friendly) or manually choose to edit the configuration file `voipd.conf` with:
```bash
vi voipd.conf
```
Configure the values corresponding to your installation :
```bash
# /data/voip/voipd.conf
# VoIP configuration for UniFiOS
#
# This file is managed by the voip-ui web interface (https://<gateway>/voip)
# You can also edit it manually. Changes take effect after restarting the service.
#
# Generated by: voip install
# ============================================================

# Physical WAN port the VoIP VLAN is attached to
# UCG-Fiber: eth4 (RJ45 WAN) or eth6 (SFP+ WAN)
# UDM-Pro:   eth8 or eth9
VOIP_WAN_INTERFACE="eth4"

# VLAN ID your ISP uses for VoIP traffic (verify with your ISP or ONT)
VOIP_WAN_VLAN="11"

# Name of the VLAN subinterface that will be created (safe to leave as "voip")
VOIP_WAN_VLAN_INTERFACE="voip"

# DSCP/QoS egress CoS mapping — maps all internal CoS values to CoS 5 (Voice)
VOIP_WAN_VLAN_INTERFACE_EGRESS_QOS="0:5 1:5 2:5 3:5 4:5 5:5 6:5 7:5"

# ISP's P-CSCF hostname for DNS resolution at startup
# Leave as example value if your ISP does not use IMS/P-CSCF
PCSCF_HOSTNAME="p-cscf.example.com"

# Routing mode: "pbr" (default, recommended) or "forward"
ROUTING_MODE="pbr"

# ── PBR mode ─────────────────────────────────────────────────────────────────
# Routing table number — auto-detected at install to avoid conflicts with
# UniFiOS WAN tables (201, 202...). Do NOT change unless you know what
# you are doing.
VOIP_RT_TABLE="203"
VOIP_RT_TABLE_NAME="voip"

# ── Forward mode (only used when ROUTING_MODE="forward") ─────────────────────
VOIP_FORWARD_INTERFACE=""
VOIP_FORWARD_GATEWAY=""

# ── IMS subnet override ───────────────────────────────────────────────────────
# Leave empty for auto-detection from DHCP-assigned IP (recommended)
# Override example: VOIP_IMS_SUBNET="10.0.0.0/8"
VOIP_IMS_SUBNET=""

# Set to "true" for verbose debug output in journalctl
VOIP_DEBUG="false"
```
Type in your keyboard `i` to enter insert mode, make your changes and then save changes and exit by hitting escape followed by `:wq` then press ENTER

### Routing modes

| Mode | Description |
|------|-------------|
| `pbr` | **Default.** Creates a dedicated routing table and ip rules. Mirrors exactly how UniFiOS handles a second WAN. No changes to existing interfaces. |
| `forward` | Routes IMS traffic through an existing interface (`VOIP_FORWARD_INTERFACE`). Use if you want to have a dedicated VLAN for IMS traffic. |

**Configuration guide:**

* `VOIP_WAN_INTERFACE`: Corresponds to the WAN interface of the gateway
  - `eth4` or `eth6` on Gateway Cloud Fiber (UCG-Fiber)
  - `eth9` on some UDM models
  - Check your specific device configuration/documentation

* `VOIP_FORWARD_INTERFACE`: The VLAN interface where VoIP traffic should be routed (where your VoIP phone/adapter is connected)
  - `br0` = default LAN bridge interface
  - `br102` = interface corresponding to VLAN 102
  - You can specify multiple interfaces separated by spaces: `"br0 br102"`

### IMS subnet detection

The script auto-detects the IMS subnet from the DHCP-assigned IP:

| First octet | Subnet used |
|-------------|-------------|
| `10.x.x.x` | `10.0.0.0/8` |
| `100.x.x.x` | `100.64.0.0/10` |
| `172.x.x.x` | `172.16.0.0/12` |
| `192.x.x.x` | `192.168.0.0/16` |
| Other | `10.0.0.0/8` (fallback) |

To override: set `VOIP_IMS_SUBNET="x.x.x.x/y"` in the config section.

Configure your SIP softphone or ATA:
- **SIP Server / Registrar**: your ISP's SIP domain
- **Outbound Proxy**: the SIP Proxy IP shown in the logs
- **SIP Username / Password**: your ISP's provided credentials
- **Transport**: UDP/TCP/TLS

Your SIP softphone or ATA should now be able to register and make/receive calls! 📞

Verify routing is correct:

```sh
# Routing table

# Policy rules (two entries at priority 100)
ip rule show | grep "100:"

ip route get <your-ISP-SIP-proxy-IP>
```

## Persistence across reboots and firmware upgrades

The `/data` directory persists across reboots and firmware upgrades on UniFiOS. The systemd service file in `/etc/systemd/system/` does **not** persist after a firmware upgrade. Reinstall it with:

```bash
cd /data/voip
./voip install
./voip install-ui
```

## Updating the VoIP Service/Web UI

To update the VoIP Service/Web UI:
```bash
cd /data/voip
./voip update
```

## Uninstallation

To remove all traces of the script:
```bash
cd /data/voip
./voip uninstall
./voip uninstall-ui
chmod +x uninstall.sh
./uninstall.sh
```

Then delete the directory:
```bash
rm -rf /data/voip
```

## Troubleshooting

### VoIP interface has no IP

```sh
ip addr show voip
journalctl -u systemd-networkd -n 30
```

The VLAN ID or WAN interface name may be wrong for your device model.

### SIP proxy not resolved

The script reads the DNS server from the DHCP lease file at `/run/systemd/netif/leases/<index>`. Verify:

```sh
IFIDX=$(ip link show voip | awk -F': ' 'NR==1{print $1}')
cat /run/systemd/netif/leases/$IFIDX
```

You should see a `DNS=` line. Then test manually:

```sh
DNS=$(grep "^DNS=" /run/systemd/netif/leases/$IFIDX | cut -d= -f2 | awk '{print $1}')
nslookup <your-pcscf-hostname> $DNS
```

### Registration times out

Monitor SIP traffic on both interfaces simultaneously:

```sh
# Terminal 1 — traffic leaving to ISP
tcpdump -i voip -n port 5060

# Terminal 2 — traffic reaching your LAN
tcpdump -i br0 -n port 5060
```

If packets appear on `voip` but replies don't appear on your LAN bridge, check conntrack:

```sh
conntrack -L 2>/dev/null | grep <SIP-proxy-IP>
```

The reply direction `dst` should be your voip IP, not your internet IP.

### Calls fail or one-way audio

Do **not** load `nf_nat_sip` or `nf_conntrack_sip`. These modules rewrite SDP in-flight and corrupt SRTP key material. Modern IMS networks use symmetric RTP and do not need SIP ALG.

```sh
# Confirm these are NOT loaded
lsmod | grep sip
# Expected: no output
```

## FAQ

**Q: What VLAN ID should I use?**  
A: See the local VLAN ID in your area, the default in this script is VLAN 11, you can verify by either checking your ISP ONT web-ui WAN page or using omci commands by telnet from your aftermarket ONT unit.

**Q: Does this support IPv6?**  
A: Currently, this script only configures IPv4. IPv6 support can be added if needed.

**Q: Will this survive a reboot?**  
A: Yes, the systemd service is enabled to start automatically on boot.
